Add `classifyPrediction(userId, _options, clientOverride)`:

1. Call `buildPrediction(userId, _options, clientOverride)`. If result is `{ sufficient: false }`, compute `daysLogged` from `storage.readExpenses(userId)`:
   - If `expenses.length === 0`, daysLogged = 0
   - Else `daysLogged = Math.floor((now.getTime() - new Date(expenses[0].timestamp).getTime()) / (24 * 60 * 60 * 1000))`
   - Return `{ sufficient: false, daysLogged }`

2. From `result.categories`, separate numeric entries (where `typeof value === 'number'`) from 'kurang data' entries.

3. Pre-populate `classifications.kost = 'tetap'` (hardcoded per roadmap decision — do NOT send kost to Claude).

4. Build a category list string for Claude: one line per numeric category (excluding kost), format: `"- makan: ~Rp 450rb"`. If no categories remain to classify (all are kost or kurang data), skip Claude call entirely and use only the pre-populated classifications.

5. Call Claude once (batch) via:
   ```javascript
   const client = clientOverride || defaultClient;
   const response = await client.messages.create({
     model: 'claude-haiku-4-5',
     max_tokens: 256,
     tools: [PREDICT_CLASSIFY_TOOL],
     tool_choice: { type: 'tool', name: 'classify_categories' },
     messages: [{ role: 'user', content: `Klasifikasikan kategori pengeluaran berikut sebagai tetap atau variabel:\n${categoryList}` }]
   });
   const block = response.content.find(b => b.type === 'tool_use' && b.name === 'classify_categories');
   ```
   Merge `block.input.classifications` into the pre-populated `classifications` map.

6. Compute variance for savings headroom (Option A — re-read expenses inside classifyPrediction):
   - Read all expenses: `const allExpenses = await storage.readExpenses(userId)`
   - Get active windows: `const windows = _selectMonthWindows(now)` then filter to active (same logic as buildPrediction)
   - For each **variable** category with a numeric estimate (not 'kurang data'):
     - Compute monthly totals for each active window where the category has at least 1 expense
     - Need at least 2 windows with data to compute variance
     - `variance = max(monthlyTotals) - min(monthlyTotals)` (max-min range)
     - `min = min(monthlyTotals)`, `avg = Math.round(sum(monthlyTotals) / monthlyTotals.length)`, `headroom = avg - min`
   - Pick the variable category with the highest `variance` value
   - If no variable category qualifies, `savings = null`
   - Otherwise `savings = { category, min, avg, headroom }`

7. Return:
   ```javascript
   {
     sufficient: true,
     monthsUsed: result.monthsUsed,
     categories: result.categories,
     classifications,
     savings  // null or { category, min, avg, headroom }
   }
   ```

Add `_formatPrediction(result)` — pure synchronous function:

1. If `result.sufficient === false`:
   Return `Data kamu baru ${result.daysLogged} hari \u2014 butuh minimal 30 hari buat prediksi yang akurat. Terus catat ya!`

2. If `result.sufficient === true`:
   - Header: `Prediksi bulan depan (berdasarkan ${result.monthsUsed} bulan terakhir):\n`
   - Empty line
   - For each category in `result.categories`:
     - Emoji from `CATEGORY_EMOJI[cat]` (fallback `'\u{1F4E6}'` for unknown)
     - If value is `'kurang data'`: `${emoji} ${cat} \u2014 kurang data`
     - If value is number: `${emoji} ${cat} \u2014 ${result.classifications[cat]} \u2014 ~Rp ${_formatAmount(value)}`
   - Empty line
   - Total line: sum all numeric category values, format as `Total kira-kira: ~Rp ${_formatAmount(total)}`
   - If `result.savings` is not null, add empty line then savings footer:
     `Kalau mau hemat, coba kurangin ${result.savings.category} \u2014 bulan lalu bisa se-rendah ${_formatAmount(result.savings.min)}, rata-ratanya ${_formatAmount(result.savings.avg)}. Ada ruang ~${_formatAmount(result.savings.headroom)} buat dihemat.`

Update `module.exports` to add `classifyPrediction`, `_formatPrediction`, `_formatAmount`.

    Replace the stub `predict.js` with the full implementation. The module structure:

    ```javascript
    'use strict';
    const storage = require('./storage');

    const WEIGHTS = { 3: [0.42, 0.33, 0.25], 2: [0.56, 0.44], 1: [1.00] };
    ```

    **`_selectMonthWindows(now)`:**
    - `now` is a Date object (defaults to `new Date()` in `buildPrediction`)
    - Loop `i = 1` to `i <= 3`: compute `month = now.getUTCMonth() - i`, `year = now.getUTCFullYear()`
    - If `month < 0`, add 12 and subtract 1 from year
    - Push `{ year, month }` to result array
    - Return the array (ordered: m-1, m-2, m-3)

    **`_filterToWindow(expenses, window)`** (internal helper, not exported):
    - Filter expenses where `new Date(e.timestamp).getUTCFullYear() === window.year && new Date(e.timestamp).getUTCMonth() === window.month`

    **`_computeCategories(expenses, windows)`:**
    - Collect all unique categories: `[...new Set(expenses.map(e => e.category))]`
    - Only consider expenses that fall within any of the given windows (filter first)
    - `n = windows.length`, `weights = WEIGHTS[n]`
    - For each category:
      - Get all expenses for that category across all windows
      - Count distinct UTC days: `new Set(catExpenses.map(e => new Date(e.timestamp).toISOString().slice(0, 10))).size`
      - If `days < 3`: set `categories[cat] = 'kurang data'` and continue
      - Otherwise: for each window `i`, sum amounts for that category in that window, multiply by `weights[i]`, accumulate
      - `categories[cat] = Math.round(estimate)`
    - Return the categories map

    **`buildPrediction(userId, _options, clientOverride)`:**
    - Extract `now` from `_options && _options.now ? _options.now : new Date()`
    - Read expenses: `const expenses = await storage.readExpenses(userId)`
    - If `expenses.length === 0`: return `{ sufficient: false }`
    - History gate: `const earliest = new Date(expenses[0].timestamp).getTime()`; if `earliest > now.getTime() - 30 * 24 * 60 * 60 * 1000`: return `{ sufficient: false }`
    - Compute windows: `const allWindows = _selectMonthWindows(now)`
    - Determine `monthsUsed`: count how many windows have at least one expense. Filter `allWindows` to only those with expenses. If `monthsUsed === 0`: return `{ sufficient: false }`
    - Compute categories using only the windows that have data (the active windows)
    - Return `{ sufficient: true, monthsUsed, categories }`

    **Important details:**
    - The history gate uses `now.getTime()` (not `Date.now()`) so tests can inject a deterministic "now" via `_options.now`
    - `_computeCategories` receives only expenses within the active windows (pre-filtered by `buildPrediction`) plus the windows array
    - Missing months for a category contribute 0 to that month's weighted term (the filter returns empty array, sum is 0)
    - `clientOverride` parameter is accepted but unused in Phase 4 — position reserved for Phase 5

    **Exports:**
